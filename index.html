Ingeniería de Sistemas - Fundamentos y Protocolos

1. Las Redes Hoy y Fundamentos

Historia y Evolución de Internet

  • Orígenes (1969): Inicia con ARPANET, una red distribuida del Departamento de Defensa de EE. UU., diseñada para resistir fallos mediante la conmutación de paquetes (rutas redundantes).
  • Unificación (1983): Se adopta el protocolo TCP/IP, conectando redes diferentes y dando nacimiento al Internet moderno.
  • Evolución (1991 - Web 2.0): Tim Berners-Lee inventa la World Wide Web (HTTP, HTML). Pasamos de la Web 1.0 (solo lectura) a la Web 2.0 (redes sociales y usuarios creadores de contenido).
  • Conectividad Móvil y Nube: La evolución abarca desde 1G hasta el 4G y 5G (baja latencia, Internet de las cosas - IoT). Surge la computación en la nube dividida en modelos: SaaS (Software), PaaS (Plataforma) e IaaS (Infraestructura) y la arquitectura híbrida.

Tipos y Topologías de Redes

Alcance Geográfico:

  • PAN: Área personal (Bluetooth, metros).
  • LAN: Área local (edificio, Ethernet/Wi-Fi).
  • MAN: Área metropolitana (ciudad, fibra óptica).
  • WAN: Área amplia (países/global, Internet).
  • SAN: Área de almacenamiento dedicada.

Topologías Físicas (Cables) vs. Lógicas (Flujo):

  • Destacan la de Bus (obsoleta, un cable central).
  • Anillo.
  • Estrella (todo pasa por un Switch central).
  • Árbol (jerárquica).
  • Malla (robusta, con rutas redundantes usada en Internet).

2. Estándares IEEE y Redes Físicas / Inalámbricas

El IEEE (Instituto de Ingenieros Eléctricos y Electrónicos) crea estándares para garantizar que equipos de distintas marcas se comuniquen entre sí (interoperabilidad). Destaca el Proyecto 802:

  • IEEE 802.3 (Ethernet): Rige redes cableadas. Evolucionó de cables coaxiales (10BASE5) a Gigabit y fibra óptica. Utiliza pares trenzados (UTP, FTP, STP) para mitigar interferencias. Las categorías varían (Cat5e hasta Cat8) en velocidad y frecuencia.
  • Conectores RJ-45: Usa 8 pines. Hay dos normas clave de colores: T568A y T568B (esta última es la más común hoy).
  • Para conectar equipos iguales (PC a PC, Switch a Switch) se usa Cable Cruzado. Para equipos distintos (PC a Switch), Cable Directo.
  • IEEE 802.11 (Wi-Fi): Redes inalámbricas mediante ondas de radio. Evolucionó por generaciones (b, g, n, ac, ax) y opera en espectros de 2.4 GHz (mayor alcance), 5 GHz (mayor velocidad) y 6 GHz. Usa encriptación WPA3 para seguridad.
  • IEEE 802.15 (PAN): Bluetooth (corto alcance, varias clases y versiones) y Zigbee.
  • IEEE 802.1Q (VLANs): Segmentación lógica de una red física para aumentar la seguridad y eficiencia.

3. Identidad de Red, DHCP y Equipos de Interconexión

El Ecosistema IP:

  • Direcciones IP (Identidad): Son etiquetas numéricas (ej. IPv4: 192.168.1.1). Pueden ser Privadas (identidad interna en LAN) o Públicas (visibles en Internet, asignadas por el ISP).
  • NAT: Mecanismo en el router que traduce la IP privada a la IP pública.
  • DHCP (Asignación): Protocolo que asigna IPs dinámicas automáticamente, simplificando la red. Opera bajo el ciclo DORA (Discover, Offer, Request, Acknowledge). Las IPs estáticas (fijas) se reservan para servidores e impresoras.
  • DNS (Traducción): Convierte nombres de dominio (ej. https://www.google.com) a direcciones IP.

Hardware y Conceptos:

Dispositivos:

  • Módem: Convierte señal análoga del ISP a digital.
  • Router: Conecta diferentes redes (LAN con Internet WAN).
  • Switch: Interconecta equipos en la misma red y manda datos solo al destinatario (usa MAC).
  • Hub: Replicaba la señal a todos generando colisiones (obsoleto).

Comandos de Diagnóstico:

  • ipconfig (ver configuración)
  • ping (conectividad básica)
  • tracert (ver saltos hacia destino)
  • netstat (conexiones activas)
  • arp (mapa de IP a MAC física).

4. Protocolos y Modelos: OSI vs TCP/IP

Para conectar equipos dispares, se estandarizó la forma en la que los datos viajan:

El Modelo OSI (Teórico / 7 Capas):

  • Física: Transmisión de bits por cables o aire (conectores, hardware).
  • Enlace de Datos: Construye Tramas (Frames), usa la MAC y detecta errores (Switches).
  • Red: Enrutamiento de Paquetes mediante direcciones IP (Routers).
  • Transporte: Entrega fiable (TCP) o rápida (UDP). Construye Segmentos y usa Puertos.
  • Sesión: Inicia, mantiene y cierra la conexión entre aplicaciones.
  • Presentación: Formato de datos (ASCII, JPEG), compresión y cifrado.
  • Aplicación: Interfaz de usuario y protocolos de uso (HTTP, FTP, SMTP).

Encapsulamiento: Cuando un emisor manda un mensaje, la información baja por las capas (de la 7 a la 1) añadiendo cabeceras; al llegar al receptor, "sube" de la 1 a la 7 quitando cabeceras.

El Modelo TCP/IP (Práctico / Real):

Es el motor actual de Internet. Condensa las 7 capas de OSI en 4 funcionales:

  • Acceso a Red (Combina Enlace y Física de OSI).
  • Internet (Equivalente a Red en OSI, usa IPv4/IPv6).
  • Transporte (TCP / UDP).
  • Aplicación (Combina Aplicación, Presentación y Sesión de OSI).
